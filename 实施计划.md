# ComfyUI 飞书机器人实施计划

## 摘要

构建一个 Python 中间服务，使用飞书企业自建应用的 Bot 长连接接收消息，调用现有 ComfyUI 服务 `http://192.168.9.101:13888` 执行本地已有 workflow，并将生成进度与最终图片回传到飞书会话。

## 关键设计

- 技术栈采用 Python，使用飞书官方 `lark-oapi` SDK 处理长连接事件，使用 `httpx` 调 ComfyUI HTTP API，使用 `websockets` 监听 ComfyUI 进度。
- workflow 来源采用本地目录：`workflows/` 存放 ComfyUI “Save API Format” 导出的 JSON。
- `workflows/index.yaml` 定义 `id`、展示名、描述、默认参数、正向提示词节点 ID、可选负向提示词节点 ID、可选 seed/width/height 节点 ID。
- 飞书交互首版使用文本命令：
  - `/画图` 返回可用 workflow 列表。
  - `/画图 <workflow_id> <提示词>` 提交任务。
  - `/状态 <task_id>` 查询任务状态。
  - `/取消 <task_id>` 删除等待任务或中断当前任务。
- 进度显示采用“发送一条任务消息并定时编辑/回复进度”的方式，默认每 5 秒最多更新一次，避免 ComfyUI 高频 WS 事件刷屏。
- 图片回传流程：`POST /api/prompt` 提交任务，`/ws?clientId=<uuid>` 监听完成事件，`GET /api/history/{prompt_id}` 找输出文件，`GET /api/view?...` 下载图片，上传到飞书图片资源后发送图片消息。

## 接口与配置

- 环境变量：
  - `FEISHU_APP_ID`
  - `FEISHU_APP_SECRET`
  - `FEISHU_BOT_NAME` 可选，用于日志标识。
  - `COMFYUI_BASE_URL=http://192.168.9.101:13888`
  - `WORKFLOW_DIR=./workflows`
  - `DATA_DIR=./data`
  - `PROGRESS_INTERVAL_SECONDS=5`
  - `MAX_CONCURRENT_TASKS=1`
- 本地模块：
  - `feishu_bot`：长连接启动、消息解析、回复消息、上传图片。
  - `comfy_client`：封装 `/api/prompt`、`/api/history/{prompt_id}`、`/api/view`、`/api/queue`、`/api/interrupt` 和 WS 事件。
  - `workflow_registry`：扫描 workflow JSON 和索引配置，校验节点 ID 是否存在。
  - `task_store`：持久化 `task_id`、`prompt_id`、飞书会话、状态、进度、输出文件和错误信息。
- 数据存储首版使用 SQLite，文件放在 `data/tasks.sqlite3`，避免进程重启后丢失任务状态。

## 实施步骤

1. 创建 Python 项目结构、依赖、配置示例、workflow 示例索引和启动入口。
2. 实现 workflow 扫描与参数注入，只允许修改配置中声明过的节点输入，避免误改 workflow 结构。
3. 实现 ComfyUI 客户端，打通提交任务、监听 WS 进度、拉取 history、下载输出图片。
4. 实现飞书长连接机器人，处理文本命令、发送任务确认、进度更新、错误回复和图片回传。
5. 实现 SQLite 任务表和后台任务调度，保证同一飞书消息对应一个可查询任务。
6. 增加接入文档：飞书应用创建、开启机器人、配置长连接、申请权限、安装到群、填写环境变量、放置 workflow、启动服务。
7. 增加最小测试与模拟脚本：workflow 配置校验、ComfyUI API mock、消息命令解析、任务状态流转。

## 测试计划

- 单元测试：
  - `/画图`、`/画图 <workflow> <prompt>`、`/状态`、`/取消` 命令解析。
  - workflow 索引加载、节点 ID 校验、提示词注入。
  - ComfyUI history 输出解析和 `/api/view` URL 生成。
- 集成测试：
  - 使用 mock ComfyUI 服务模拟 `queued -> running -> progress -> succeeded`。
  - 模拟 ComfyUI `execution_error`，确认飞书错误回复内容稳定。
  - 模拟飞书图片上传失败，确认任务仍标记完成但返回上传错误。
- 手工验收：
  - 在飞书群发送 `/画图` 能看到 workflow 列表。
  - 发送 `/画图 <workflow_id> 一只赛博朋克猫` 后，群内出现任务确认、进度更新和最终图片。
  - ComfyUI 不可达时，机器人不崩溃并返回可理解的错误。
  - `/取消 <task_id>` 对等待任务能删除，对运行任务能调用 `/api/interrupt`。

## 假设与默认值

- 飞书接入使用企业自建应用的 Bot 长连接，不使用普通 webhook 机器人。
- 首版只支持文本到图；输入图、遮罩图、多轮参数编辑和交互式卡片放到后续版本。
- 首版一次消息只提交一个 ComfyUI 任务；默认并发 1，避免占满显存。
- workflow 必须由 ComfyUI 导出为 API 格式 JSON。
- ComfyUI 服务与机器人服务网络互通，机器人机器可以访问 `http://192.168.9.101:13888`。
